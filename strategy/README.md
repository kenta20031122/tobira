# Tobira Strategy Workspace

This directory is the shared working area for product and marketing strategy.
It is designed so multiple subagents can work in parallel without drifting.

## Recommended order

1. Read `00-context.md`
2. Confirm or edit `01-assumptions.md`
3. Lock the funnel in `02-funnel.md`
4. Record decisions in `03-decisions.md`
5. Read `05-initial-strategy-draft.md`
6. Read `06-channel-plan.md`
7. Use `06-subagent-briefs.md` or `04-research-questions.md` to assign work
8. Convert the strategy into execution in `07-experiments.md`

## Ground rules

- Optimize for the next 90 days first.
- Keep the primary ICP to one segment at a time.
- Separate current facts from hypotheses.
- Every recommendation should point to one KPI.
- If a subagent proposes a major change, log it in `03-decisions.md`.

## Suggested subagent split

- Product strategy: activation, retention, monetization, roadmap focus
- Marketing strategy: ICP, channels, messaging, content system
- Research: competitors, pricing, user pains, JTBD
- Measurement: event taxonomy, funnel definitions, dashboard specs

## Working mode

- Facts come from the product, codebase, analytics, or direct user research.
- Hypotheses must include a validation method.
- Decisions should include a re-evaluation trigger.
