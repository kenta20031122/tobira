# スポット追加マニュアル

## 全体フロー

```
1. 追加スポットを決める
2. SQLファイルを作成（scripts/seed-{地域}-extra.sql）
3. Supabase SQL Editor or Claude に実行させる
   → この時点では is_published = false のため本番ユーザーには見えない
4. ファクトチェック（Claude の japan-tourism-fact-checker エージェント）
5. 修正が必要な箇所を apply-factcheck-{地域}.sql で UPDATE
6. fact_checked_at を更新
7. 写真を追加する（npm run review-photos / import-photo.mjs 等）
   → 写真アップロード完了時に is_published = true になりユーザーに公開される
```

---

## ステップ1: スポットを決める

まず既存スポットを確認して重複を避ける。

```sql
-- 都道府県別の現在のスポット数
SELECT prefecture, COUNT(*) as count
FROM spots
GROUP BY prefecture
ORDER BY count ASC;

-- 特定都道府県のスポット一覧
SELECT id, name FROM spots WHERE prefecture = 'Hokkaido' ORDER BY name;
```

## ステップ2: SQL ファイル作成

`scripts/seed-{地域}-extra.sql` として保存。

### 全フィールド一覧

| フィールド | NULL許可 | 説明 |
|-----------|---------|------|
| `id` | NO | kebab-case、英語（例: `hakodate-morning-market`） |
| `name` | NO | 英語名 |
| `prefecture` | NO | 例: `'Hokkaido'`、`'Tokyo'`（[型定義](#prefecture-一覧)参照） |
| `region` | NO | 例: `'hokkaido'`（[地域一覧](#region-一覧)参照） |
| `categories` | NO | `ARRAY['nature', 'history']`（[カテゴリ一覧](#category-一覧)参照） |
| `description` | NO | 英語、3〜5文 |
| `address` | NO | 英語住所（例: `'Wakamatsucho, Hakodate, Hokkaido'`） |
| `lat` | NO | 緯度（小数点4桁） |
| `lng` | NO | 経度（小数点4桁） |
| `image_url` | NO | Unsplash プレースホルダーでよい（[下記参照](#image_url)） |
| `tags` | NO | `ARRAY['tag1', 'tag2']`、5〜7個 |
| `is_premium` | NO | `true` = ホームページに表示候補 |
| `highlights` | NO | `ARRAY['...', '...', '...', '...']`、**4項目** |
| `best_season` | NO | 例: `'June–October'` |
| `access` | NO | 最寄り駅・バスのアクセス方法（英語） |
| `admission` | NO | 料金（無料なら `'Free'`） |
| `duration` | NO | 例: `'2–3 hours'` |
| `website_url` | NO | 公式サイト URL（不明なら `''`） |
| `opening_hours` | YES | 営業時間（例: `'9:00–17:00 (closed Mondays)'`） |
| `tips` | YES | 訪問者へのアドバイス（1〜3文） |
| `is_published` | NO | **INSERT 時は省略でよい（DEFAULT false）**。写真アップロード時に自動で `true` になる |

> ⚠️ `opening_hours` と `tips` は NULL 許可だが、**必ず入れること**。
> 古いシードスクリプトにはこれらがないが、DB にはカラムがあり UI にも表示される。

> ℹ️ `is_published = false` のスポットはユーザーに表示されない。写真スクリプト（`import-photo.mjs` 等）が自動で `true` に変更する。手動で公開したい場合は `UPDATE spots SET is_published = true WHERE id = 'spot-id';`

### SQLテンプレート

```sql
INSERT INTO spots (
  id, name, prefecture, region, categories, description,
  address, lat, lng, image_url, tags, is_premium,
  highlights, best_season, access, admission, duration,
  website_url, opening_hours, tips
) VALUES
(
  'spot-id',
  'Spot Name',
  'Prefecture',
  'region',
  ARRAY['nature'],
  'Description text (3–5 sentences).',
  'Address, City, Prefecture',
  35.0000, 135.0000,
  'https://images.unsplash.com/photo-1540959733332-eab4deabeeaf?w=800&auto=format&fit=crop&q=80',
  ARRAY['tag1', 'tag2', 'tag3', 'tag4', 'tag5'],
  false,
  ARRAY[
    'Highlight 1.',
    'Highlight 2.',
    'Highlight 3.',
    'Highlight 4.'
  ],
  'April–October',
  'Access description from nearest station.',
  'Free',
  '2–3 hours',
  'https://example.com',
  '9:00–17:00 (closed Mondays)',
  'Tips for visitors.'
),
-- 複数スポットを続ける場合はカンマで繋ぐ、最後はセミコロン
;
```

### image_url

Unsplash プレースホルダーはそのまま使う。後で実際の画像に置き換え可。


## ステップ3: DB に反映

### Claude に頼む場合
「実行して」と言えば Supabase MCP で自動実行。

### 自分で実行する場合
Supabase Dashboard → **SQL Editor** に SQL を貼り付けて Run。

### 確認

```sql
-- 追加されたか確認
SELECT id, name, opening_hours IS NOT NULL AS has_hours, tips IS NOT NULL AS has_tips
FROM spots
WHERE prefecture = '対象都道府県'
ORDER BY created_at DESC
LIMIT 20;
```

## ステップ4: ファクトチェック

Claude に以下のように依頼する：

```
scripts/seed-{地域}-extra.sql のスポットをファクトチェックして
```

`japan-tourism-fact-checker` エージェントが日本語の権威あるソースをもとに
admission / access / opening_hours の誤りを検出してレポートを出す。

レポートは `scripts/spot-factcheck-report-{地域}.md` に保存される。

---

## ステップ5: ファクトチェック結果を適用

修正が必要なスポットは `scripts/apply-factcheck-{地域}.sql` を作成してUPDATE。

```sql
-- apply-factcheck-{地域}.sql のテンプレート
UPDATE spots SET
  admission = '¥500 (adult), free under 12',
  access = '10-min walk from XX Station'
WHERE id = 'spot-id';

-- 複数フィールド一括更新
UPDATE spots SET opening_hours = v.opening_hours, tips = v.tips
FROM (VALUES
  ('spot-id-1', '9:00–17:00', 'Tip text.'),
  ('spot-id-2', '10:00–18:00', 'Another tip.')
) AS v(id, opening_hours, tips)
WHERE spots.id = v.id;

-- チェック完了フラグ
UPDATE spots SET fact_checked_at = NOW()
WHERE prefecture = '対象都道府県' AND fact_checked_at IS NULL;
```

## よくあるミス

| ミス | 対策 |
|------|------|
| `opening_hours` / `tips` を入れ忘れる | 古いシードスクリプトを参考にしない。このテンプレートを使う |
| `id` が既存と被る | 事前に `SELECT id FROM spots WHERE id = 'xxx'` で確認 |
| `prefecture` のスペルミス | 上の Prefecture 一覧からコピペ |
| `highlights` が4項目でない | ちょうど4つ入れる（UIが4枚グリッド想定） |
| 座標が大幅にずれている | Google Maps で確認してから入力 |
| `website_url` を空にする | `''`（空文字）を入れる。NULL はNG（NOT NULL 制約あり） |
| INSERT 直後にユーザーに見えてしまう | 気にしなくてOK。`is_published` は DEFAULT false なので写真追加まで非表示 |
| 写真追加したのに表示されない | `SELECT is_published FROM spots WHERE id = 'xxx'` で確認。false なら `UPDATE spots SET is_published = true WHERE id = 'xxx'` |
