# 写真追加マニュアル

## 通常フロー（Pexels + Wikimedia 自動取得）

```bash
# 1. 写真が必要なスポットを確認
npm run list-unimaged

# 2. 候補写真を自動取得（Pexels + Wikimedia Commons）
npm run fetch-candidates

# 3. 候補を見ながらキー1つで承認・アップロード
npm run review-photos -- --upload
```

### review-photos の操作キー

| キー | 動作 |
|---|---|
| `1`〜`5` | その番号の候補を承認してアップロード |
| `p` | 全候補NG → Pixtaキューに追加 |
| `s` | スキップ（次回も出てくる） |
| `o` | Finder で候補フォルダを開く |
| `q` | 終了 |

---

## Pixta フロー（品質重視・ニッチなスポット）

### まとめてインポート（複数枚）

```bash
# 1. Pixtaで写真をダウンロード
# 2. ~/Downloads/pixta-import/ フォルダに移動
# 3. 一括割り当て・アップロード
npm run batch-import -- ~/Downloads/pixta-import/
# 4. 作業フォルダ削除
rm -rf ~/Downloads/pixta-import/
```

### 1枚だけインポート

```bash
node scripts/import-photo.mjs <spot-id> ~/Downloads/pixta_xxxxx.jpg
```

## 写真ソース

| ソース | APIキー | クレジット表示 | 向いているスポット |
|---|---|---|---|
| Pexels | `PEXELS_API_KEY`（無料） | 不要 | 定番観光地 |
| Wikimedia Commons | 不要 | CC BY系は自動表示 | 有名スポット |
| Pixta | なし（手動DL） | 不要 | ニッチ・高品質が必要な場所 |

---

## Pixtaキューの確認

`npm run list-unimaged` で 🔴Pixta マークのスポットが確認できる。

```bash
npm run list-unimaged
# → lake-mashu  [Hokkaido] nature  mashu, hokkaido 🔴Pixta
```

## スクリプト一覧

| コマンド | 用途 |
|---|---|
| `npm run list-unimaged` | 写真未設定スポット一覧 |
| `npm run fetch-candidates` | Pexels/Wikimediaから候補取得 |
| `npm run review-photos -- --upload` | インタラクティブ承認・アップロード |
| `npm run batch-import -- <フォルダ>` | Pixtaの複数枚を一括インポート |
| `node scripts/import-photo.mjs <id> <file>` | 1枚インポート |
