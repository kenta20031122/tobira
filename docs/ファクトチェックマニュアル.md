# ファクトチェックマニュアル

スポットの **入場料・アクセス・営業時間・tips** を公式情報で検証し、誤りがあれば DB を更新する手順です。

---

## 全体フロー

```
1. 未チェックのスポットを地域別に取得（fetch-unchecked-spots.mjs）
2. japan-tourism-fact-checker で4項目を検証し、報告書を作成
3. 報告書から apply-factcheck-<地域>.sql を生成（extract-factcheck-sql.mjs）
4. Supabase で SQL を実行し、fact_checked_at を更新して完了
```

**検証する4項目:** `admission`（入場料） / `access`（アクセス） / `opening_hours`（営業時間） / `tips`（アドバイス）

## ステップ1: 未チェックスポットの取得

```bash
cd tobira
node scripts/fetch-unchecked-spots.mjs <region>
```

例: `node scripts/fetch-unchecked-spots.mjs okinawa`

- `scripts/factcheck-<region>.json` に保存される
- 標準出力にも JSON が出る（エージェントに貼り付ける用）
- **取得項目:** id, name, prefecture, address, access, admission, opening_hours, tips, best_season, website_url, description

未チェックが 0 件の地域では空の配列になります。進捗確認は次で:

```sql
SELECT prefecture, COUNT(*) AS total,
  COUNT(fact_checked_at) AS checked,
  COUNT(*) - COUNT(fact_checked_at) AS remaining
FROM spots
GROUP BY prefecture
ORDER BY prefecture;
```

---

## ステップ2: エージェントで検証

1. **Cursor で japan-tourism-fact-checker を起動**
2. **渡すもの:** ステップ1の JSON（`factcheck-<region>.json` の内容または標準出力のコピー）
3. **依頼内容の例:**
   - 「この JSON のスポットについて、admission / access / opening_hours / tips の4項目を公式・一次情報で検証し、報告書を書いてください」
   - 「報告書の末尾に ## SQL更新候補 を付け、修正が必要な行だけ `| id | field | 修正後の値（英文）|` の表で出してください。field は admission, access, opening_hours, tips のいずれか」
4. **報告書の保存先:** `scripts/spot-factcheck-report-<region>.md`（または batch 付き）

判定は **✅ 確認済み / ❌ 誤り / ⚠️ 要注意 / ❓ 未確認** のいずれかで統一してもらうとよいです。

---

## ステップ3: 適用用 SQL の生成

報告書ができたら、表から UPDATE 文を生成します。

```bash
cd tobira
echo "-- 参照: spot-factcheck-report-<region>.md" > scripts/apply-factcheck-<region>.sql
node scripts/extract-factcheck-sql.mjs scripts/spot-factcheck-report-<region>.md >> scripts/apply-factcheck-<region>.sql
```

例（沖縄の場合）:

```bash
echo "-- 参照: spot-factcheck-report-okinawa.md" > scripts/apply-factcheck-okinawa.sql
node scripts/extract-factcheck-sql.mjs scripts/spot-factcheck-report-okinawa.md >> scripts/apply-factcheck-okinawa.sql
```

**末尾にチェック済みフラグを手動で追加:**

```sql
UPDATE spots SET fact_checked_at = NOW()
WHERE prefecture IN ('Okinawa') AND fact_checked_at IS NULL;
```

対象県は地域に合わせて複数書きます（例: 関西なら `'Osaka','Kyoto','Hyogo','Nara','Shiga','Wakayama'`）。

---

## ステップ4: Supabase で実行

1. Supabase Dashboard → SQL Editor を開く
2. `scripts/apply-factcheck-<region>.sql` の**中身全体**をコピーして貼り付け
3. Run で実行

※ 必ず **.sql ファイル** を開くこと。`.mjs` などのスクリプトを実行するとエラーになります。

実行方法の詳細は **docs/SQL-RUN-SETUP.md** を参照。

## トラブルシュート

- **「取得件数: 0」**  
  `fact_checked_at IS NULL` のスポットがその地域にない状態。別地域を指定するか、進捗確認クエリで残りを確認。

- **SQL 実行で syntax error at "#!/"**  
  .sql ではなく .mjs を開いて実行している。必ず `apply-factcheck-<region>.sql` を開いて中身をコピーすること。

- **報告書に SQL更新候補がない**  
  エージェントに「修正が必要な行だけ、表形式で ## SQL更新候補 に書く」と再度指示する。
