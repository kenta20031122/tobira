# ブログ記事作成マニュアル

## 概要

Tobira のブログ記事は Supabase の `blog_posts` テーブルに SQL で INSERT して公開する。
記事には2種類あり、`article_type` で判定して異なるレイアウトを表示する。

| `article_type` | 内容 | レイアウト |
|---|---|---|
| `generic` | 複数エリア横断のリスト系記事（例: Best places beyond Tokyo） | 標準レイアウト（セクション一覧） |
| `prefecture` | 特定都道府県を深掘りするガイド記事（例: Hokkaido guide） | HERO → QuickFacts + 地図 → CHARACTER → sections → SpotMap → HowToGetThere → More Prefectures |

---

## 全体フロー（共通）

```
STEP 1  対象都道府県のスポットを DB から取得・確認（必須）
   ↓
STEP 2  DBスポットをもとに場所・構成をユーザーに確認
   ↓
STEP 3  SQL INSERT 文を生成
   ↓
STEP 4  Supabase SQL Editor で実行
   ↓
STEP 5  カバー画像をアップロード（Storage）
   ↓
STEP 6  cover_image を UPDATE
   ↓
STEP 7  本番で確認
   ↓
STEP 8  blog-article-prompt.md の既存記事リストを更新
```

---

## STEP 1 — DB確認（必須）

**構成を決める前に、必ず対象都道府県のスポットをDBから取得すること。**
DBにないスポットは `spot_query` で何も表示されないため、記事の品質に直結する。

```js
// Supabase JS クライアントで確認する場合の例
// ⚠️ address の ilike だけだと "Kamikochi" が "Kochi" にマッチするなど誤ヒットが起きる
// → prefecture カラムで絞り込むのが確実
const { data } = await supabase
  .from('spots')
  .select('name, address, categories')
  .eq('prefecture', 'Fukuoka')   // 対象都道府県の英語表記に変更（大文字始まり）
  .order('name');
```

またはターミナルから:
```bash
cd tobira
node -e "
require('dotenv').config({ path: '.env.local' });
const { createClient } = require('@supabase/supabase-js');
const sb = createClient(process.env.NEXT_PUBLIC_SUPABASE_URL, process.env.SUPABASE_SERVICE_ROLE_KEY);
// prefecture で絞る（ilike '%Kochi%' だと Kamikochi 等が誤ヒットするため）
sb.from('spots').select('id, name, address, categories').eq('prefecture', 'Fukuoka').order('name')
  .then(({ data }) => { console.log('Total:', data.length); data.forEach(s => console.log(s.id, '|', s.name, '|', s.address, '|', JSON.stringify(s.categories))); });
"
```

⚠️ **注意: `ilike '%Kochi%'` のような address 部分一致は誤ヒットの原因になる。**
例: `Kamikochi`（長野）が `%Kochi%` にマッチしてしまう。
**必ず `prefecture` カラムの完全一致（`.eq('prefecture', '...')`）で絞ること。**

取得したスポット一覧をもとにセクション構成を決める。
DBにスポットが少ないエリアはセクションを統合するか除外する。

---

## STEP 2 — 構成確認

SQL を書く前に必ず以下の形式でユーザーに確認を取ること:

```
以下の構成でよいですか？確認後に SQL を生成します。

1. [場所名] — [都道府県] — [一言説明]
2. ...
```

チェックポイント:
- 同じ都道府県の場所が2つ以上ない（1記事1県まで）
- 既存記事と全く同じ場所の組み合わせでない
- セクション数が8以下（closing 除く）
- 各セクションの `address_contains` がDBのスポット住所と一致することを確認済み

---

## STEP 3 — SQL 生成

### generic タイプ

```sql
insert into blog_posts (
  slug, title, description, read_minutes, cover_image,
  intro, sections, cta_heading, cta_body, related_region,
  article_type, status, published_at
) values (
  'your-slug',
  'Your Title',
  'SEO meta description. 140–160 chars.',
  8,
  null,
  'Intro paragraph (2–3 sentences).',
  '[...sections JSON...]'::jsonb,
  'CTA heading',
  'CTA body text.',
  null,
  'generic',
  'published',
  now()
);
```

### prefecture タイプ

generic の全カラムに加えて以下を追加する:

```sql
insert into blog_posts (
  slug, title, description, read_minutes, cover_image,
  intro, sections, cta_heading, cta_body, related_region,
  article_type,
  character,          -- その都道府県の「性格」を語る文章（3〜5文）
  quick_facts,        -- JSONB: { access, best_season, vibe_tags }
  how_to_get_there,   -- 交通アクセスの詳細（新幹線・飛行機・フェリーなど）
  status, published_at
) values (
  'hokkaido-travel-guide',
  'Hokkaido: Japan''s Northern Frontier',
  'SEO description...',
  12,
  null,
  'Intro text.',
  '[...sections JSON...]'::jsonb,
  'CTA heading',
  'CTA body.',
  'hokkaido',
  'prefecture',
  'Character text (3–5 sentences). The personality of the place.',
  '{"access": "Direct flights from Tokyo (Haneda/Narita) to Sapporo (New Chitose) take 1.5 hrs.", "best_season": "Feb for snow festivals · Jun–Aug for lavender · Oct for autumn colour", "vibe_tags": ["wild", "outdoors", "food-forward", "wide open spaces", "slow travel"]}'::jsonb,
  'How to get there text. Include shinkansen, flight, ferry options as applicable.',
  'published',
  now()
);
```

## sections JSONB 構造

### generic / prefecture 共通セクション

```json
[
  {
    "heading": "セクション見出し（場所名 + 視点を込める）",
    "travel_time": "2.5 hrs from Tokyo by Shinkansen",
    "best_season": "Apr–May for cherry blossoms · Oct–Nov for autumn colour",
    "body": "段落1\n\n段落2\n\n段落3",
    "spot_query": {
      "address_contains": "Otaru",
      "prefecture": "Hokkaido",
      "categories": ["history"],
      "limit": 3
    },
    "section_link": {
      "text": "Explore Hokkaido spots",
      "href": "/guides/hokkaido"
    }
  },
  {
    "heading": "How to Choose",
    "body": "締めの文章（spot_query・section_link は不要）"
  }
]
```

**フィールド補足:**

| フィールド | 必須 | 説明 |
|---|---|---|
| `heading` | ✅ | 場所名だけでなく視点・角度を込める |
| `body` | ✅ | 段落を `\n\n` で区切る。各段落200〜300文字 |
| `travel_time` | brief次第 | 主要都市からのアクセス時間 |
| `best_season` | 任意 | 省略するとDBのspot値が自動表示 |
| `spot_query` | 任意（closing以外推奨） | DBクエリで自動スポット取得 |
| `section_link` | `spot_query`と同時に | `href`は `/guides/[prefecture]` 形式 |

**`spot_query` 注意点:**
- `address_contains` は部分一致（ilike）なので必ず `prefecture` を併用する
- "Ise" → "Daisen" にもマッチするリスクあり → `"prefecture": "Mie"` で防ぐ
- 会津若松は `"address_contains": "Aizu"` を使う（"Aizu-Wakamatsu" はマッチしないことがある）
- `categories`: `nature` / `history` / `onsen` / `food` / `activity` / `spiritual`

---

## quick_facts JSONB 構造（prefecture タイプのみ）

```json
{
  "access": "Direct flights from Tokyo (Haneda/Narita) to Sapporo (New Chitose) take around 1.5 hours. The Hokuto limited express connects Sapporo to Hakodate in 4 hours.",
  "best_season": "Feb for snow festivals · Jun–Aug for lavender · Oct for autumn colour",
  "vibe_tags": ["wild", "outdoors", "food-forward", "wide open spaces", "slow travel"]
}
```

| フィールド | 内容 |
|---|---|
| `access` | 東京などの主要都市からの主要アクセス手段（1〜2文） |
| `best_season` | ベストシーズン（`·` 区切りで複数可） |
| `vibe_tags` | その場所の雰囲気を表す3〜6個のタグ（小文字・ハイフン区切り） |

---

## character フィールド（prefecture タイプのみ）

その都道府県の「性格」を語る段落。観光情報ではなく、**その場所がどんな空気感・文化・感触を持つか**を主観的・opinionated に書く。3〜5文。

例（北海道）:
```
Hokkaido does not feel like the rest of Japan. There is space here — real space — between towns, between mountains, between ideas about what a journey should be. The pace is slower, the seasons more extreme, and the food more honest: dairy, seafood, ramen built for cold weather. People come once for the novelty of snow or lavender fields, and return because Hokkaido rewards those who slow down.
```

---

## how_to_get_there フィールド（prefecture タイプのみ）

交通アクセスの詳細。以下を含める:
- 主要手段（飛行機・新幹線・フェリー）とおおよその所要時間
- 現地での移動手段（鉄道・バス・レンタカーの必要性）

---

## ライティングスタイル（全タイプ共通）

- 自信を持った編集スタイル。リスト記事のような書き方はしない
- 具体的：実際の地名・詳細・正直な評価
- British English（travelling, colour, favour, centre）
- **禁止ワード:** nestled / breathtaking / must-visit / hidden gem
- セクション見出しは場所名だけでなく視点・角度を込める
  - 例: "Kanazawa — The Kyoto Nobody Told You About"
  - 例: "Beppu — Where the Earth Is Still Boiling"
  - 例: "Hokkaido — Where Japan Finally Has Room to Breathe"

---

## SQL 生成時の注意

- 文中の `'`（シングルクォート）は `''`（2個）でエスケープする
- body の改行は `\n` で表現する
- sections / quick_facts は末尾に `::jsonb` キャストをつける

**出力前チェックリスト:**
- [ ] 禁止ワード未使用
- [ ] British English
- [ ] cover_image は `null`（INSERT時）
- [ ] article_type を設定（generic or prefecture）
- [ ] prefecture タイプ: character / quick_facts / how_to_get_there が全て入っている
- [ ] spot_query に address_contains + prefecture + categories が全て入っている
- [ ] シングルクォートを `''` でエスケープ済み
- [ ] sections / quick_facts に `::jsonb` 付き

## STEP 4 — Supabase で実行

Supabase Dashboard → **SQL Editor** に INSERT 文を貼り付けて実行。

## STEP 5 & 6 — カバー画像

**Storage パス:** `spot-images/blog/`

**アップロード後の URL 形式:**
```
https://khgpsvnrorfigvubxhmd.supabase.co/storage/v1/object/public/spot-images/blog/{ファイル名}
```

**UPDATE 文:**
```sql
update blog_posts
set cover_image = '{画像URL}'
where slug = '{SLUG}';
```
